### 1.2 数据流向

```
普通API请求流程：
手机 → Cloudflare(443) → 阿里云Nginx(80) → 后端服务(3000) → SQLite数据库 → 返回

AI请求流程：
手机 → Cloudflare(443) → Cloudflare Tunnel → 本地大模型(8000) → 返回

SSH连接流程：
本地电脑 → Cloudflare(443) → Cloudflare SSH Tunnel → 阿里云SSH(22) → 返回
```

### 1.3 端口占用一览

| 端口 | 服务 | 位置 | 是否对外暴露 | 说明 |
|------|------|------|-------------|------|
| 22 | SSH | 阿里云 | ❌ 仅Workbench | 已通过安全组限制 |
| 80 | Nginx | 阿里云 | ✅ Cloudflare代理 | HTTP入口 |
| 443 | HTTPS | Cloudflare | ✅ 公网 | Cloudflare SSL终结 |
| 3000 | Node.js后端 | 阿里云 | ❌ 仅本地 | Nginx反向代理 |
| 8000 | Qwen大模型 | 本地电脑 | ❌ 仅本地 | Cloudflare Tunnel暴露 |
| 8080 | cloudflared metrics | 本地电脑 | ❌ 仅本地 | Tunnel监控 |

## 二、域名和DNS配置

### 2.1 域名信息

| 域名 | 用途 | 指向 | 备注 |
|------|------|------|------|
| <YOUR_DOMAIN> | 主域名（登录/注册/数据） | 阿里云 <YOUR_SERVER_IP> | 普通API |
| www.<YOUR_DOMAIN> | www子域名 | 阿里云 <YOUR_SERVER_IP> | 可选 |
| api.<YOUR_DOMAIN> | AI大模型 | Cloudflare Tunnel → localhost:8000 | AI功能 |
| ssh.<YOUR_DOMAIN> | SSH远程管理 | Cloudflare SSH Tunnel → localhost:22 | 远程管理 |

### 2.2 Cloudflare DNS 记录（详细）

**位置：** Cloudflare Dashboard → DNS → Records

| 类型 | 名称 | 内容 | 代理状态 | TTL | 说明 |
|------|------|------|----------|-----|------|
| A | @ | <YOUR_SERVER_IP> | 开启（橙色云朵） | Auto | 主域名指向阿里云 |
| A | www | <YOUR_SERVER_IP> | 开启（橙色云朵） | Auto | www子域名 |
| CNAME | api | <YOUR_TUNNEL_ID>.cfargotunnel.com | 开启（橙色云朵） | Auto | AI大模型穿透 |
| CNAME | ssh | <YOUR_SSH_TUNNEL_ID>.cfargotunnel.com | 开启（橙色云朵） | Auto | SSH穿透 |

### 2.3 DNS配置步骤

**步骤1：添加A记录（阿里云服务器）**
```
1. 登录 Cloudflare Dashboard
2. 选择 <YOUR_DOMAIN> 域名
3. 点击 DNS → Records → Add record
4. 填写：
   - Type: A
   - Name: @ （代表根域名）
   - IPv4 address: <YOUR_SERVER_IP>
   - Proxy status: 开启（橙色云朵）
   - TTL: Auto
5. 点击 Save
6. 重复步骤，添加 www A记录
```

**步骤2：添加CNAME记录（Cloudflare Tunnel）**
```
1. 点击 DNS → Records → Add record
2. 填写：
   - Type: CNAME
   - Name: api
   - Target: <YOUR_TUNNEL_ID>.cfargotunnel.com
   - Proxy status: 开启（橙色云朵）
   - TTL: Auto
3. 点击 Save
4. 重复步骤，添加 ssh CNAME记录
```

### 2.4 DNS验证命令

```bash
# 测试域名解析
nslookup <YOUR_DOMAIN>
nslookup api.<YOUR_DOMAIN>
nslookup ssh.<YOUR_DOMAIN>

# 测试HTTPS访问
curl -I https://<YOUR_DOMAIN>
curl -I https://api.<YOUR_DOMAIN>
curl -I https://ssh.<YOUR_DOMAIN>

# 测试HTTP访问（应该被重定向到HTTPS）
curl -I http://<YOUR_DOMAIN>
```

### 2.5 DNS常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 域名无法解析 | DNS记录未生效 | 等待5分钟，或检查记录是否正确 |
| 解析到错误IP | DNS缓存 | 等待TTL过期，或本地刷新DNS |
| 代理状态错误 | 橙云/灰云选择错误 | 橙云=Cloudflare代理，灰云=仅DNS |

## 三、SSL/TLS配置

### 3.1 配置位置

**位置：** Cloudflare Dashboard → SSL/TLS → Overview

### 3.2 当前配置

| 配置项 | 值 | 说明 |
|--------|-----|------|
| SSL/TLS encryption mode | **Flexible** | 最重要的配置 |
| Always Use HTTPS | **On** | HTTP自动跳转HTTPS |
| HTTP Strict Transport Security (HSTS) | **On** | 安全头 |
| Minimum TLS Version | **TLS 1.2** | 最低TLS版本 |
| Opportunistic Encryption | **On** | 自动加密 |
| Automatic HTTPS Rewrites | **On** | 自动修复混合内容 |

### 3.3 为什么选择Flexible模式

```
访问流程：
手机 → Cloudflare(HTTPS:443) → 阿里云Nginx(HTTP:80) → 后端(3000)

原因：
1. 本地大模型服务是HTTP（不是HTTPS）
2. 阿里云服务器没有SSL证书
3. Cloudflare负责SSL终结（对用户HTTPS，对源站HTTP）
4. Flexible模式：Cloudflare通过HTTP访问源站
```

**⚠️ 重要：** 如果选择 Full 或 Full (Strict)，Cloudflare会尝试通过HTTPS访问源站，但源站没有SSL证书，会导致521错误。

### 3.4 SSL/TLS验证命令

```bash
# 测试HTTPS证书
curl -vI https://<YOUR_DOMAIN> 2>&1 | grep -E "SSL|certificate|issuer"

# 测试HTTP到HTTPS重定向
curl -I http://<YOUR_DOMAIN>

# 测试TLS版本
curl --tlsv1.2 -I https://<YOUR_DOMAIN>
curl --tlsv1.3 -I https://<YOUR_DOMAIN>
```

## 四、Cloudflare Tunnel配置

### 4.1 Tunnel概述

本项目使用两个Cloudflare Tunnel：

| Tunnel名称 | 用途 | 本地端口 | 对外域名 |
|------------|------|---------|---------|
| qwen-tunnel | AI大模型 | localhost:8000 | api.<YOUR_DOMAIN> |
| ssh-tunnel | SSH管理 | localhost:22 | ssh.<YOUR_DOMAIN> |

### 4.2 Tunnel配置步骤

**步骤1：安装cloudflared（本地电脑）**
```bash
# Windows（推荐用winget）
winget install cloudflare.cloudflared

# 或手动下载
# https://developers.cloudflare.com/cloudflare-one/connections/connect-networks/downloads/
```

**步骤2：创建Tunnel（Cloudflare Dashboard）**
```
1. 登录 Cloudflare Dashboard
2. 点击 Zero Trust → Networks → Tunnels
3. 点击 "Create a tunnel"
4. 选择 "Cloudflared"
5. 给Tunnel命名（如：qwen-tunnel）
6. 点击 "Create tunnel"
7. 复制 Token（保存好，启动时需要）
```

**步骤3：配置Tunnel路由**
```
1. 在 Tunnel 配置页面
2. 点击 "Add route" 或 "Public Hostnames"
3. 添加路由：
   - Subdomain: api
   - Domain: <YOUR_DOMAIN>
   - Service Type: HTTP
   - URL: localhost:8000
4. 点击 "Save"
```

**步骤4：启动Tunnel（本地电脑）**
```bash
# 方法1：使用Token启动
cloudflared tunnel run --token <你的token>

# 方法2：使用配置文件启动（高级）
cloudflared tunnel run qwen-tunnel
```

### 4.3 Tunnel配置文件（高级）

**配置文件位置：** `~/.cloudflared/config.yml`

```yaml
tunnel: <tunnel-id>
credentials-file: ~/.cloudflared/<tunnel-id>.json

ingress:
  - hostname: api.<YOUR_DOMAIN>
    service: http://localhost:8000
  - service: http_status:404
```

### 4.4 Tunnel验证命令

```bash
# 测试本地大模型
curl http://localhost:8000

# 测试Tunnel是否正常
curl https://api.<YOUR_DOMAIN>

# 查看Tunnel状态
cloudflared tunnel info qwen-tunnel
```

### 4.5 Tunnel常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| Tunnel无法连接 | Token过期或错误 | 重新创建Tunnel，获取新Token |
| 域名无法访问 | 路由未配置 | 检查Tunnel的Public Hostnames配置 |
| 521错误 | 源站服务未运行 | 检查本地大模型是否启动 |
| 连接超时 | 本地防火墙 | 检查Windows防火墙设置 |

### 5.3 服务器安全加固（详细）

#### 5.3.1 Nginx反向代理 + 限流

**配置文件：** `/etc/nginx/conf.d/00-rate-limit.conf`

```nginx
# 普通API限流：5次/秒
limit_req_zone $binary_remote_addr zone=api_limit:10m rate=5r/s;

# 登录接口限流：1次/秒
limit_req_zone $binary_remote_addr zone=login_limit:10m rate=1r/s;

# AI接口限流：2次/秒
limit_req_zone $binary_remote_addr zone=ai_limit:10m rate=2r/s;

# 连接数限制
limit_conn_zone $binary_remote_addr zone=conn_limit:10m;
```

**配置文件：** `/etc/nginx/sites-available/default`

```nginx
server {
          listen 80 default_server;
    server_name _;

    include /etc/nginx/snippets/cloudflare-realip.conf;
    include /etc/nginx/snippets/security.conf;

    # 登录接口
    location /api/auth/ {
        limit_req zone=login_limit burst=3 nodelay;
        limit_conn conn_limit 5;

        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_redirect off;
    }

    # AI接口（/v1/是大模型API路径）
    location /v1/ {
        limit_req zone=ai_limit burst=5 nodelay;
        limit_conn conn_limit 10;

        proxy_pass http://127.0.0.1:3000;
        proxy_read_timeout 300s;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_redirect off;
    }

    # 普通API
    location /api/ {
        limit_req zone=api_limit burst=10 nodelay;
        limit_conn conn_limit 20;

        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_redirect off;
    }

    # 其他请求返回403
    location / {
        return 403;
    }
}
```

#### 5.3.2 fail2ban防暴力破解

**配置文件：** `/etc/fail2ban/jail.local`

```ini
[DEFAULT]
bantime = 3600  # 默认1小时
findtime = 600  # 10分钟内
maxretry = 3    # 3次失败
backend = auto
ignoreip = 127.0.0.1/8 ::1  # 忽略本地

[sshd]
enabled = true
port = ssh
filter = sshd
logpath = /var/log/auth.log
maxretry = 3
bantime = 86400  # SSH暴力破解封24小时

[nginx-limit-req]
enabled = true
port = http,https
filter = nginx-limit-req
logpath = /var/log/nginx/error.log
maxretry = 5
bantime = 7200  # Nginx限流触发封2小时

[nginx-botsearch]
enabled = true
port = http,https
filter = nginx-botsearch
logpath = /var/log/nginx/access.log
maxretry = 2
bantime = 86400  # 扫描行为封24小时
```

## 八、多设备管理与数据同步

### 8.1 功能概述

系统支持多设备管理，核心规则：
- **同一账号最多同时登录 1 台手机 + 1 台 PC**
- 同设备类型再次登录时，旧设备自动被踢出并收到提示
- AI 识别结果可跨设备同步（未采纳的结果从旧设备转到新设备）

### 8.2 数据库表结构

#### login_sessions 表（设备登录管理）

```sql
CREATE TABLE login_sessions (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL,
  token TEXT NOT NULL,
  device_type TEXT NOT NULL CHECK(device_type IN ('mobile', 'pc')),
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES instructors(id) ON DELETE CASCADE
);
CREATE INDEX idx_login_sessions_user ON login_sessions(user_id);
CREATE INDEX idx_login_sessions_token ON login_sessions(token);
```

#### pending_ai_results 表（跨设备 AI 结果同步）

```sql
CREATE TABLE pending_ai_results (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  user_id INTEGER NOT NULL,
  results TEXT NOT NULL,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (user_id) REFERENCES instructors(id) ON DELETE CASCADE
);
CREATE INDEX idx_pending_ai_user ON pending_ai_results(user_id);
```

### 8.3 踢出机制原理

```
新设备登录流程：
1. 新设备发送 POST /auth/login（带 deviceType: 'mobile' 或 'pc'）
2. 服务端 DELETE FROM login_sessions WHERE user_id=? AND device_type=?
   → 旧设备的 token 立即失效
3. 服务端 INSERT 新 token 到 login_sessions
4. 返回新 token + pendingAiResults（旧设备未采纳的 AI 结果）

旧设备被踢流程：
1. 旧设备的下一个 API 请求（或 10 秒定时器请求）
2. authMiddleware 查 login_sessions 找不到旧 token
3. 返回 401 → 前端弹出"账号已在别处登录"提示框
4. 清除本地 token，跳转登录页
```

### 8.4 前端踢出检测机制

#### request.js（请求层 401 拦截）

```javascript
let _kickShown = false;

function request(url, method = 'GET', data = {}, timeout = 60000) {
  return new Promise((resolve, reject) => {
    // 空 token 保护：未登录时直接跳登录页
    if (!app.globalData.token && !url.includes('/auth/')) {
      wx.reLaunch({ url: '/pages/login/login' });
      return reject(new Error('未登录'));
    }
    wx.request({
      url: app.globalData.baseUrl + url,
      method, data, timeout,
      header: {
        'Content-Type': 'application/json',
        'Authorization': 'Bearer ' + app.globalData.token
      },
      success(res) {
        if (res.data.code === 401) {
          // 立即清除 token，防止后续请求继续使用失效 token
          app.globalData.token = '';
          wx.removeStorageSync('token');
          if (!_kickShown) {
            _kickShown = true;
            wx.showModal({
              title: '账号已在别处登录',
              content: '当前有另外一台设备登录了此账号，您已被迫下线。',
              showCancel: false,
              confirmText: '重新登录',
              success() {
                _kickShown = false;
                wx.reLaunch({ url: '/pages/login/login' });
              }
            });
          } else {
            wx.reLaunch({ url: '/pages/login/login' });
          }
          return reject(new Error('未登录'));
        }
        resolve(res.data);
      },
      fail(err) {
        wx.showToast({ title: '网络错误', icon: 'none' });
        reject(err);
      }
    });
  });
}
```

#### app.js（10 秒定时器兜底检测）

### 8.6 AI 结果跨设备同步流程

```
旧设备 → 新设备 AI 结果同步流程：

1. 旧设备：教员输入评语 → AI 识别 → 结果显示在面板上（未采纳）
2. 旧设备：结果同时保存到：
   a. 本地 storage（key: aiResults_<sessionId>）
   b. 服务器 pending_ai_results 表（POST /assessments/pending-ai）
3. 新设备登录 → 服务端返回 pendingAiResults
4. 新设备：将 pendingAiResults 存入 pending_ai_import
5. 新设备：进入记录面板 → importPendingResults() 检测到待导入数据
6. 新设备：自动显示 AI 结果面板，教员勾选后采纳
```

### 8.7 路由顺序（重要！）

`assessments.js` 中，`/pending-ai` 路由必须在 `/:sessionId` **前面**，否则 Express 会把 `"pending-ai"` 当作 sessionId 参数匹配：

```javascript
// ✅ 正确顺序
router.post('/pending-ai', ...);    // 先匹配固定路径
router.get('/pending-ai', ...);
router.delete('/pending-ai', ...);
router.get('/check-session', ...);
router.get('/:sessionId', ...);     // 后匹配参数路径
router.post('/:sessionId', ...);

// ❌ 错误顺序（会导致 pending-ai 被当作 sessionId）
router.get('/:sessionId', ...);     // "pending-ai" 被匹配为 sessionId
router.post('/pending-ai', ...);    // 永远不会被执行
```

### 8.8 authMiddleware（异步版本）

```javascript
const jwt = require('jsonwebtoken');
const db = require('../config/db');

const JWT_SECRET = process.env.JWT_SECRET || '<YOUR_JWT_SECRET>';

async function authMiddleware(req, res, next) {
  const token = req.headers.authorization?.replace('Bearer ', '');
  if (!token) {
    return res.status(401).json({ code: 401, msg: '未登录' });
  }
  try {
    const decoded = jwt.verify(token, JWT_SECRET);
    // 验证 token 是否存在于 login_sessions 表
    const [sessions] = await db.query(
      'SELECT id FROM login_sessions WHERE token = ? AND user_id = ?',
      [token, decoded.id]
    );
    if (sessions.length === 0) {
      return res.status(401).json({ code: 401, msg: '登录已过期' });
    }
    req.user = decoded;
    next();
  } catch {
    return res.status(401).json({ code: 401, msg: '登录已过期' });
  }
}

module.exports = { authMiddleware, JWT_SECRET };
```

## 十一、故障排查大全

### 问题1：手机登录/注册失败

**检查清单：**
1. 微信公众平台 request 合法域名是否包含 `https://<YOUR_DOMAIN>`

**测试命令：**

```bash
curl https://<YOUR_DOMAIN>/api/health
# 预期：{"status":"ok"}
```

**解决步骤：**
```bash

sudo systemctl status nginx
sudo nginx -t

# 2. 检查后端服务
pm2 status
pm2 logs icao-server

# 3. 检查安全组（阿里云控制台）
# 确认只允许Cloudflare IP段
```

### 问题2：AI功能失败

**检查清单：**
1. 本地大模型是否运行（端口8000）
2. Cloudflare Tunnel是否运行
3. Tunnel路由是否配置正确

**测试命令：**
```bash
# 测试本地
curl http://localhost:8000

# 测试Tunnel
curl https://api.<YOUR_DOMAIN>
```

**常见原因：**
| 原因 | 症状 | 解决方案 |
|------|------|----------|
| 大模型未启动 | localhost:8000 无响应 | 双击start-qwen.bat |
| Tunnel未启动 | api.<YOUR_DOMAIN> 521错误 | 双击qwen_tunnel_run.bat |
| Token过期 | Tunnel连接失败 | 重新创建Tunnel获取新Token |
| 防火墙拦截 | 本地正常但外网不行 | 检查Windows防火墙 |

### 问题3：SSH连接失败

**检查清单：**
1. 本地Cloudflare SSH Tunnel是否运行
2. SSH config是否正确
3. 私钥文件是否正确
4. 服务器SSH配置是否正确

**测试命令：**
```bash
ssh -v ssh.<YOUR_DOMAIN>
```

**常见错误：**
| 错误信息 | 原因 | 解决方案 |
|---------|------|----------|
| "Permission denied" | 私钥不匹配 | 检查服务器公钥是否与本地私钥匹配 |
| "Connection refused" | SSH服务未运行 | 通过Workbench检查sshd服务 |
| "Connection timed out" | Tunnel未运行 | 启动SSH Tunnel |
| "Host key verification failed" | 首次连接 | 输入yes确认 |

### 问题7：Workbench登录失败（安全加固后）

**原因：** SSH禁用了root登录或密码登录。

**解决方法：**
1. 通过阿里云VNC控制台登录
2. 临时修改SSH配置：
   ```bash
   sudo sed -i 's/PermitRootLogin no/PermitRootLogin yes/' /etc/ssh/sshd_config
   sudo sed -i 's/PasswordAuthentication no/PasswordAuthentication yes/' /etc/ssh/sshd_config
   sudo systemctl reload sshd
   ```
3. 登录后及时恢复安全配置

### 问题8：登录后显示"登录中"然后无反应

**原因：** 服务器上的 `middleware/auth.js` 未更新，`authMiddleware` 导出格式不对。

**检查方法：**
```bash
# 查看服务器日志
sudo pm2 logs icao-server --lines 30

# 如果看到以下错误：
# TypeError: Router.use() requires a middleware function
#   at Object.<anonymous> (/var/www/icao-assessment/server/routes/sessions.js:5:8)
# 说明 middleware/auth.js 未正确导出
```

**解决：**
```bash
# 确认服务器上的 middleware/auth.js 内容正确
cat /var/www/icao-assessment/server/middleware/auth.js

# 应该包含：
# async function authMiddleware(req, res, next) { ... }
# module.exports = { authMiddleware, JWT_SECRET };

# 如果内容不对，重新上传本地的 middleware/auth.js
# 然后重启
sudo pm2 restart icao-server
```

1. 小程序未更新（旧版没有 10 秒定时器）

## 十二、易错点总结

### 12.1 DNS相关易错点

| # | 易错点 | 正确做法 |
|---|--------|----------|
| 1 | A记录的IP写错 | 必须是阿里云公网IP：<YOUR_SERVER_IP> |
| 2 | CNAME记录类型选错 | api和ssh必须是CNAME，不是A |
| 3 | 橙色云朵未开启 | 所有记录必须开启代理（橙色云朵） |
| 4 | www记录遗漏 | 需要单独添加www的A记录 |
| 5 | TTL设置过长 | 保持Auto即可 |

### 12.2 SSL/TLS相关易错点

| # | 易错点 | 正确做法 |
|---|--------|----------|
| 1 | 模式选了Full | 必须选Flexible（源站无SSL证书） |
| 2 | HSTS未开启 | 开启后强制HTTPS |
| 3 | 未开启Always Use HTTPS | 开启后HTTP自动跳转 |

### 12.3 Cloudflare Tunnel相关易错点

| # | 易错点 | 正确做法 |
|---|--------|----------|
| 1 | Token过期未更新 | Token过期后需重新创建Tunnel |
| 2 | 路由未配置 | Tunnel必须配置Public Hostnames |
| 3 | 本地大模型未启动就开Tunnel | 先启动大模型，再启动Tunnel |
| 4 | 关了Tunnel命令行窗口 | 窗口必须保持打开状态 |
| 5 | Tunnel和SSH Tunnel混淆 | 两个Tunnel的Token不同，不能混用 |

### 12.4 阿里云服务器易错点

| # | 易错点 | 正确做法 |
|---|--------|----------|
| 1 | 安全组规则方向搞错 | 一定是"入方向"规则 |
| 2 | 端口范围写错 | 是1/65535不是1-65535 |
| 3 | SSH规则开放太大 | 只允许100.104.0.0/16（Workbench） |
| 4 | 禁用了root后无法登录 | 确保wmx用户已创建且有sudo权限 |
| 5 | 禁用了密码登录后无法登录 | 确保已上传公钥到服务器 |
| 6 | node_modules从Windows复制 | 必须在Linux上重新npm install |
| 7 | pm2用start而不是restart | 服务已在运行时用restart |

### 12.5 微信小程序易错点

| # | 易错点 | 正确做法 |
|---|--------|----------|
| 1 | 只加了一个域名 | 必须同时加<YOUR_DOMAIN>和api.<YOUR_DOMAIN> |
| 2 | 域名未备案 | 域名必须先备案才能添加 |
| 3 | http://而不是https:// | 必须用https:// |
| 4 | 漏了api子域名 | AI功能需要api.<YOUR_DOMAIN> |

### 12.6 本地电脑易错点

| # | 易错点 | 正确做法 |
|---|--------|----------|
| 1 | bat文件窗口闪退 | 确保文件末尾有pause |
| 2 | 中文乱码 | bat文件使用英文或UTF-8编码 |
| 3 | 模型路径错误 | 确认E:\GGUF_MODELS下有模型文件 |
| 4 | SSH config路径错误 | 必须在<YOUR_HOME_DIR>\.ssh\config |
| 5 | 私钥权限问题 | 私钥文件不能被其他用户读取 |

### 12.7 多设备管理易错点

| # | 易错点 | 正确做法 |
|---|--------|----------|
| 1 | middleware/auth.js 未更新 | 必须导出 `{ authMiddleware, JWT_SECRET }`，函数必须是 async |
| 2 | assessments.js 路由顺序错误 | `/pending-ai` 必须在 `/:sessionId` 之前 |
| 3 | 迁移脚本在 ~ 目录运行 | 必须 cd 到 `/var/www/icao-assessment/server` 再运行 |
| 4 | 登录时 loggingIn 标志未重置 | finally 块中必须 `app.globalData.loggingIn = false` |
| 5 | 踢出定时器干扰登录 | app.js 中定时器必须检查 `loggingIn` 标志 |
| 6 | model 名称被修改 | AI 模型名必须保持 `'qwen'`，不要改成完整模型名 |

## 十六、常用运维命令速查

### 阿里云服务器

```bash
# 连接服务器
ssh ssh.<YOUR_DOMAIN>

# 查看服务状态
pm2 status
sudo systemctl status nginx
sudo systemctl status fail2ban

# 查看日志
pm2 logs icao-server
sudo tail -f /var/log/nginx/access.log
sudo tail -f /var/log/nginx/error.log

# 重启服务
sudo /var/www/start-server.sh
# 或分别重启
sudo systemctl restart nginx
pm2 restart icao-server

# 测试Nginx配置
sudo nginx -t

# 查看fail2ban状态
sudo fail2ban-client status
sudo fail2ban-client status sshd

# 解封IP
sudo fail2ban-client set sshd unbanip <IP>

# 查看系统资源
free -h
df -h
top

# 数据库迁移（升级后执行）
cd /var/www/icao-assessment/server
sudo node scripts/migrate-login-sessions.js
sudo node scripts/migrate-pending-ai.js
sudo pm2 restart icao-server

# 首次部署（初始化数据库）
cd /var/www/icao-assessment/server
npm install
node scripts/init-sqlite.js
pm2 start app.js --name icao-server
```

### 更新服务端文件流程

```bash
# 1. 通过 SSH 上传更新的文件（在本地电脑执行）
scp server/middleware/auth.js ssh.<YOUR_DOMAIN>:/var/www/icao-assessment/server/middleware/
scp server/routes/auth.js ssh.<YOUR_DOMAIN>:/var/www/icao-assessment/server/routes/
scp server/routes/assessments.js ssh.<YOUR_DOMAIN>:/var/www/icao-assessment/server/routes/

# 2. SSH 登录服务器
ssh ssh.<YOUR_DOMAIN>

# 3. 运行迁移
cd /var/www/icao-assessment/server
sudo node scripts/migrate-login-sessions.js
sudo node scripts/migrate-pending-ai.js

# 4. 重启服务
sudo pm2 restart icao-server

# 5. 验证
curl -X POST https://<YOUR_DOMAIN>/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"phone":"<YOUR_PHONE>","password":"<YOUR_PASSWORD>","deviceType":"mobile"}'
```
